# Governance Charter and RACI Model

## Purpose

This document defines governance ownership for security and compliance decisions
in LCP.

## Governance charter model

Each governance charter should define:

- mission and scope
- decision authority boundaries
- mandatory review cadence
- escalation and exception path
- required governance artifacts

## Role model

| Role | Core responsibility | Primary artifacts |
| --- | --- | --- |
| Security governance owner | Approves security strategy and control direction | Governance charter, annual objectives |
| Compliance manager | Maintains compliance plan and audit readiness | Compliance plan, evidence map |
| Service owner | Implements controls for assigned services | Service control implementation records |
| Operations lead | Ensures day-to-day control execution | Operational review logs, action tracking |
| Internal assurance lead | Verifies control effectiveness | Assurance reports, findings register |

## RACI template

Use the following ownership model for governance activities:

| Activity | Security governance owner | Compliance manager | Service owner | Operations lead | Internal assurance lead |
| --- | --- | --- | --- | --- | --- |
| Define control objectives | A | R | C | C | C |
| Approve policy changes | A | R | C | C | I |
| Implement service controls | C | C | A/R | R | I |
| Review risk treatment | A | R | C | C | C |
| Validate control effectiveness | I | C | C | C | A/R |

Legend: `R` Responsible, `A` Accountable, `C` Consulted, `I` Informed.

## Minimum governance cadence

- monthly governance checkpoint for open risks and exceptions
- quarterly control and policy review
- annual governance charter review
