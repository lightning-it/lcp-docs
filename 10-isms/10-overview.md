# ISMS Overview

## Purpose

This document defines the scope and intent of the ISMS layer in LCP.

The ISMS component establishes the governance baseline that all services and
delivery workflows must follow.

## Core objectives

- define accountable security ownership
- establish policy-driven governance
- implement risk-based decision making
- ensure audit-ready traceability and evidence

## Scope

The ISMS layer covers:

- governance model and responsibilities
- policy and control objective structure
- risk management and treatment lifecycle
- audit-readiness and evidence expectations

## Operating model in LCP

- `10-isms/` defines governance, risk, and assurance expectations
- `20-pgf/` translates those expectations into operational processes
- `30-modulix/` implements technical controls through automation

## Expected outcomes

- consistent governance across all services
- documented and reviewable security decisions
- clear ownership for control implementation
- repeatable internal and external audit preparation
