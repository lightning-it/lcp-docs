# Architecture Big Picture

ModuLix is organized around one execution contract and multiple documentation layers:

1. Runtime contract (`scripts/ansible-nav`) in `modulix-automation`
2. Execution containers (toolbox + ansible EE + devtools EE)
3. Service playbooks and service specs
4. Operator runbooks
5. Development and troubleshooting guides

Use this architecture section as entry point by perspective:

- Containers: `20-containers.md`
- Services: `30-services.md`
- Runbooks: `40-runbooks.md`
- Development: `50-development.md`
- Troubleshooting: `60-troubleshooting.md`
